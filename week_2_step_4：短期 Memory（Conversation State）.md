目标一句话：

👉 让 Agent **记住自己刚做过什么、为什么这么做、接下来该避免什么**

这一步完成后，你的 Agent 会明显更**稳定、连贯、少走回头路。**

**一、为什么现在一定要加 Memory？**

你现在的 Agent 已经能：

+ 给多个方案

+ 系统裁决

+ 有 FSM

但它**仍然会：**

+ 重复用同一个失败工具

+ 忘记刚刚被拒绝的方案

+ 在多步任务中“绕圈”

👉原因只有一个：

**Context ≠ Memory**

**二、短期 Memory 的工程定义（非常重要）**

我们先不碰向量库（那是第 4 周）。

**短期 Memory = 结构化的“运行态信息”**，包括：

+ 已执行过的 Action

+ 最近的 Observation

+ 被拒绝的候选

+ 失败原因

+ 当前子目标（Sub-goal）

⚠️ 关键点

**短期 Memory 不是给人看的，是给 Agent 用的**
