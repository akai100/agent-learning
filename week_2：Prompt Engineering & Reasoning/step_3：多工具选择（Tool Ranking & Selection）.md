**目标一句话：**

👉 让模型**给出多个可行动作方案**，由系统**评估并选择最优执行**

从这一刻起，你的 Agent 不再是“单线程反应器”，而是：

**具备“方案空间 + 决策权”的执行系统**

**一、为什么「单工具 → 多工具」是质变**

**现在你已有的能力**

+ 模型：提出 一个 Action

+ 系统：校验 → 执行 / 拒绝

**现实世界的问题**

+ 很多任务**有多种做法**

+ 有的：

  + 更快但不安全

  + 更安全但更慢

  + 更便宜但不准确

👉 单 Action = 赌运气

👉 多 Action = 可决策

**二、核心思想（一定要记住）**

我们把 ReAct 升级为：

```yaml
Thought
Action Candidates:
1. tool_a: ...
2. tool_b: ...
3. NONE
```

然后：

+ ❌ 模型不执行

+ ✅ 系统 评分 + 选择

⚠️ Agent 工程的铁律：
**模型给“可能性”，系统做“决定”**
