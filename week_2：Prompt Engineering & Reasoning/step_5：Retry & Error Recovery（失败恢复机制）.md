**目标一句话：**

👉 让 Agent 在 **工具失败 / 解析失败 / 决策失败** 时

**有策略地重试，而不是盲目循环**

**一、为什么“失败恢复”这么重要？**

真实世界里：

+ API 会超时

+ 工具会抛异常

+ 模型会输出不合规格式

+ Router 会拒绝 Action

❌ 没有恢复机制的 Agent = Demo

✅ 有恢复机制的 Agent = 系统

**二、失败不是一种状态，是一个「分支」**

我们引入一个核心理念：

**Error ≠ End，而是一个 State**

**三、设计升级（非常小，但非常关键）**

我们加 3 个能力：

**1. Retry State**

**2. 失败类型分类**

**3. 有限次数重试**
